# 온프레미스 vs 온디바이스 — 어디서 도느냐

## 한 줄 요약
둘 다 **"데이터를 제3자 클라우드로 안 보낸다"**는 철학은 같지만, **연산이 도는 위치**가 다르다: **온프레미스 = 내(조직) 서버**, **온디바이스 = 각 사용자 기기**.

## 공통점
- 데이터를 **외부 클라우드로 내보내지 않는다** → 프라이버시·규제·오프라인에 유리. 둘 다 **클라우드 의존의 반대 방향**.

## 핵심 차이
- **[[온프레미스]]** — "**클라우드냐 내 서버냐**"(인프라 **소유**)의 문제. **서버가 존재**하고, 사용자는 내부망으로 **접속**한다(클라이언트-서버).
- **[[온디바이스]]** — "**서버냐 단말이냐**"(연산 **위치**)의 문제. **서버가 아예 없고**, 사용자 기기가 곧 실행 주체(오프라인 가능).

## 비교표
| 기준 | 온프레미스 | 온디바이스 |
|---|---|---|
| 연산 위치 | 조직의 **중앙 서버**(내부망) | **개별 사용자 기기**(폰·노트북·엣지) |
| 규모·성능 | 강력·다수 사용자 서빙 | 제한적·1기기 |
| 네트워크 | 서버 **접속 필요**(내부망) | **없어도 됨**(오프라인) |
| 소유·운영 | 조직이 인프라 소유 | (기기는) 사용자 |
| 반대 개념 | **클라우드**(인프라 임대) | 클라우드/**서버 추론** |

## AI로 보면 (제일 명확)
- **온프레미스 LLM** — 회사가 **사내 GPU 서버**에 모델을 올려([[vLLM]]) 직원들이 **사내망 API**로 호출. 서버는 있고 여러 명이 공유. → [[로컬 LLM과 셀프호스팅|셀프호스팅]]
- **온디바이스 LLM** — **각자의 폰/노트북**에서 소형 모델이 **직접** 추론(Gemini Nano, 노트북 Ollama). 서버 없음, 데이터가 기기 밖으로 안 나감. → [[온디바이스 모델 배포]]

## 언제 무엇을
- **온프레미스** — 규제 산업·사내 대용량 데이터·**다수 사용자** 서빙·강력한 GPU가 필요할 때.
- **온디바이스** — **개인 프라이버시**·저지연·오프라인·**서버비 0**, 기기에서 충분히 도는 소형 모델일 때.

## 주의 — 완전 대립이 아니라 '다른 축'
- 온프레미스는 **인프라 소유** 축, 온디바이스는 **연산 위치** 축이라 서로 직교한다.
- 스펙트럼으로 보면: **클라우드(남의 서버) ↔ 온프레미스(내 서버) ↔ 온디바이스(서버 없이 기기)** — 뒤로 갈수록 "데이터가 더 가까이, 통제는 더 강하게, 대신 자원은 더 제한".
