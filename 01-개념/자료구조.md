# 자료구조 (Data Structure)

## 한 줄 정의
데이터를 **효율적으로 저장·접근·관리**하기 위해 정해 둔 구조. 무엇을 자주 하느냐에 따라 알맞은 구조가 다르다.

## 자세히
- **배열(Array)** — 같은 타입을 **연속된 메모리**에 저장. 인덱스로 즉시 접근(O(1)). 크기가 고정적.
- **연결 리스트(Linked List)** — 각 노드가 데이터 + "다음 노드 주소"를 가짐. 삽입·삭제가 유연(중간 삽입 쉬움), 임의 접근은 느림.
- **스택(Stack)** — **LIFO**(나중에 넣은 게 먼저 나옴). 함수 호출, 되돌리기(undo), 괄호 검사.
- **큐(Queue)** — **FIFO**(먼저 넣은 게 먼저 나옴). 작업 대기열, [[메시지 큐]], 너비우선탐색(BFS).
- **해시 테이블(Hash Table)** — `키 → 해시함수 → 위치`로 매핑. 평균 O(1)의 빠른 조회·삽입·삭제. (파이썬 dict, JS object)
- **트리(Tree)** — 계층 구조. 이진 탐색 트리(BST)는 정렬·탐색에 강함. 파일시스템·[[데이터베이스 인덱스|DB 인덱스(B-tree)]].
- **그래프(Graph)** — 노드 + 엣지로 **관계**를 표현. 길찾기(최단경로), SNS 관계망.

## 왜 중요한가
- "어떤 자료구조를 고르느냐"가 곧 성능([[시간 복잡도]])을 결정한다.
- 코딩 테스트·실무 성능 최적화 모두의 기본기.
