# HTTP / HTTPS

## 한 줄 정의
웹에서 클라이언트(브라우저)와 서버가 **요청-응답으로 대화하는 규칙(프로토콜)**이 HTTP, 거기에 **암호화(TLS)를 입힌** 게 HTTPS.

## 자세히
- **요청-응답 구조**: 클라이언트가 요청(request)을 보내면 서버가 응답(response)을 준다. 한 번 주고받으면 끝나는 **무상태(stateless)** 프로토콜이라, 상태 유지는 [[쿠키 세션 JWT|쿠키·세션]] 등으로 따로 해야 한다.
- **HTTP 메서드**: 무엇을 할지 나타낸다.
  - `GET`(조회) · `POST`(생성) · `PUT`(전체 수정) · `PATCH`(부분 수정) · `DELETE`(삭제).
- **상태 코드(status code)**: 결과를 숫자로 알려주며 5가지 계열로 나뉜다.
  - **1xx** 정보 / **2xx** 성공(`200 OK`, `201 Created`) / **3xx** 리다이렉트(`301`, `302`) / **4xx** 클라이언트 잘못(`400`, `401`, `403`, `404`) / **5xx** 서버 잘못(`500`, `503`).
- **HTTPS = HTTP over TLS**: 본문을 **TLS(과거 SSL)**로 암호화해 도청·변조·위장을 막는다. 인증서(certificate)로 서버 신원도 확인한다.
- **포트**: HTTP는 기본 **80번**, HTTPS는 기본 **443번** 포트를 쓴다. 주소는 각각 `http://`, `https://`로 시작.

## 왜 중요한가
- 모든 웹·[[REST API]]·앱 통신의 바탕. 상태 코드와 메서드는 백엔드 개발의 공용어다.
- 오늘날 HTTPS는 사실상 기본값 — 검색 순위·브라우저 경고·보안 모두 HTTPS를 전제로 한다.

## 관련 개념
- [[REST API]] — HTTP 메서드·상태 코드 위에 세운 API 설계 스타일
- [[TCP-IP|TCP/IP]] — HTTP가 그 위에서 도는 하위 전송 계층
- [[쿠키 세션 JWT]] — 무상태 HTTP에서 로그인 상태를 유지하는 방법
- [[암호화와 해싱]] — HTTPS의 TLS가 쓰는 암호화 원리
