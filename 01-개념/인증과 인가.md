# 인증 vs 인가 (Authentication vs Authorization)

## 한 줄 정의
**인증(Authentication)**은 "너 누구야?"를 확인하는 것, **인가(Authorization)**는 "너 이거 해도 돼?"를 확인하는 것.

## 자세히
- **인증(AuthN)** — 신원 확인. 아이디·비밀번호, OTP, 생체인증 등으로 "이 사람이 맞다"를 증명한다.
- **인가(AuthZ)** — 권한 확인. 인증된 사용자가 **특정 자원·기능에 접근할 권한**이 있는지 본다(역할/권한, RBAC 등).
- **순서**: 보통 인증이 먼저, 인가가 그다음. (로그인 → 그 사람이 관리자 메뉴를 볼 수 있는지 판단)
- HTTP 상태 코드로 구분된다:
  - **401 Unauthorized** — *인증* 실패. "네가 누군지 모르겠다." → 로그인하면 해결.
  - **403 Forbidden** — *인가* 실패. "누군지는 알지만 권한이 없다." → 다시 로그인해도 소용없음.
  - (이름이 헷갈리게 붙어 있다: 401의 단어는 'Unauthorized'지만 실제로는 *인증* 문제다.)

## 왜 중요한가
- 모든 로그인·권한 시스템의 기본 뼈대. 둘을 혼동하면 보안 구멍이나 잘못된 에러 응답이 나온다.
- 401/403을 올바로 구분해 내려주는 것은 API 설계의 기본기.
