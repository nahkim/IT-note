# PDF 콘텐츠 연산자 (Content Stream Operators)

## 한 줄 정의
[[PDF 구조와 QDF|PDF]] 페이지의 **콘텐츠 스트림 안에서 "무엇을 그릴지" 지시하는 명령어**. 피연산자를 먼저 쓰고 연산자를 뒤에 두는 **후위(postfix) 표기**다.

## 자세히
- 콘텐츠 스트림은 `피연산자… 연산자` 의 나열이다. 예: `72 700 Td` → (72, 700)이 피연산자, `Td`가 연산자. (계산기 RPN과 같은 방식)
- **대소문자 관습**: 보통 **소문자 = 칠하기(non-stroking/fill)**, **대문자 = 선(stroking)**. 예: `rg`(채움색 RGB) vs `RG`(선색 RGB).

### 그래픽 상태(Graphics State)
- **`q` / `Q`** — 상태 저장 / 복원(스택). 변환·색을 임시로 바꿨다 되돌릴 때 짝으로 쓴다.
- **`cm`** — 현재 변환행렬(CTM) 곱하기(이동·확대·회전). **`w`** — 선 두께.

### 경로(Path) 구성 → 칠하기
- 구성: **`m`**(moveto, 시작) · **`l`**(lineto) · **`c`**(3차 베지에) · **`re`**(사각형) · **`h`**(닫기).
- 칠하기: **`S`**(선 그리기) · **`f`**(채우기) · **`B`**(채움+선) · **`n`**(그리지 않고 경로 종료).

### 색(Color)
- **`rg`/`RG`**(RGB) · **`g`/`G`**(그레이) · **`k`/`K`**(CMYK) · `cs`/`CS`·`scn`/`SCN`(색공간 지정).

### 텍스트(Text) — 반드시 `BT … ET` 안에서만
- 블록: **`BT`**(텍스트 시작) / **`ET`**(끝).
- 상태: **`Tf`**(폰트·크기) · `Tc`(자간) · `Tw`(어간) · `TL`(줄간격) · `Tr`(렌더 모드).
- 위치: **`Td`**(다음 줄 시작 offset) · `TD` · **`Tm`**(텍스트 행렬) · `T*`(다음 줄).
- 출력: **`Tj`**(문자열 출력) · **`TJ`**(위치 조정 배열로 출력) · `'`·`"`(줄바꿈 후 출력).

### 외부 객체 / 이미지
- **`Do`** — XObject(이미지·재사용 폼)를 그린다. (인라인 이미지는 `BI … ID … EI`)

## 왜 중요한가
- "PDF 안의 글자/도형은 어떻게 그려지나"의 실제 정답 — 텍스트 추출·워터마크·도장 삽입·렌더링 디버깅의 기본.
- [[PDF 구조와 QDF|QDF]]로 콘텐츠 스트림을 펼쳐 보면 이 연산자들이 그대로 보여, **직접 읽고 수정**할 수 있다.

## 관련 개념
- [[PDF 구조와 QDF]] — 연산자가 담기는 콘텐츠 스트림과 전체 파일 구조
- [[선형화 PDF]] — 같은 콘텐츠를 웹 스트리밍용으로 재배치한 변형
- [[컴파일러와 인터프리터]] — 뷰어가 연산자를 한 줄씩 해석해 그리는 점이 인터프리터와 닮음
