# AI 에이전트 (AI Agent)

## 한 줄 정의
[[LLM]]이 스스로 **목표를 세우고 도구를 호출하며 여러 단계를 반복**해 작업을 끝까지 수행하는 시스템.

## 자세히
- 단순 챗봇은 "한 번 묻고 한 번 답"하지만, 에이전트는 **계획 → 도구 사용 → 결과 관찰 → 다시 계획**의 루프를 돈다.
- 핵심 부품: **도구(tool / function calling)**, **메모리**, **계획(planning)**, 외부 지식용 [[RAG]].
- 도구·데이터 소스를 표준 방식으로 붙이기 위한 프로토콜 **MCP(Model Context Protocol)**가 사실상 표준으로 자리잡는 중.
- 예: 코딩 에이전트(Claude Code), 개인 비서 에이전트(Gemini Spark), 업무 자동화 에이전트.
- 한계: 단계가 길어질수록 **오류 누적**·비용 증가·무한 루프 위험 → 검증·권한 제어 같은 **가드레일**이 필요하다.

## 왜 중요한가
- 2026년 AI의 핵심 흐름. 무게중심이 "답을 **생성**"에서 "일을 **수행**"으로 이동 중. (→ [[2026-06-10]] 뉴스)
- 기업용 에이전트(관측성·접근제어·컴플라이언스) 플랫폼이 쏟아지고 있다.
