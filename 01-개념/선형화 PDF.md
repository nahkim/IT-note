# 선형화 PDF (Linearized PDF / Fast Web View)

## 한 줄 정의
큰 PDF를 **전부 내려받기 전에 첫 페이지부터 바로 보여줄 수 있게** 객체 배치를 재구성한 PDF. Acrobat의 "빠른 웹 보기(Fast Web View)"가 이것.

## 자세히
- 일반 [[PDF 구조와 QDF|PDF]]는 핵심 색인(xref)·`/Root`가 **파일 끝**에 있어, 원칙적으로 끝까지 받아야 페이지를 그릴 수 있다. 선형화는 이 한계를 푼다.
- 재구성 방식:
  - 맨 앞에 **선형화 파라미터 사전(Linearization Parameter Dictionary)**을 둔다(파일 길이·페이지 수·첫 페이지 xref 위치·힌트 스트림 오프셋을 담음).
  - 그 바로 뒤에 **1페이지 렌더링에 필요한 객체**(페이지 사전·콘텐츠·폰트·이미지)를 **연속 배치**.
  - **xref가 둘**: 앞쪽의 **첫-페이지 xref** + 파일 끝의 **메인 xref**.
  - **힌트 스트림 2개**: **페이지 오프셋 힌트 테이블**(각 페이지 객체의 바이트 범위)·**공유 객체 힌트 테이블**(여러 페이지가 공유하는 객체).
- 동작: 뷰어가 앞부분만 받아 1페이지를 그리고, 사용자가 뒤 페이지로 가면 **힌트 테이블을 보고 그 부분만** 서버에 요청한다.
- **전제 조건**: 서버가 **HTTP Range 요청(byte-serving)**을 지원해야 효과가 난다(대부분 최신 서버는 기본 지원). 파일을 통째로 내려받는 상황에선 이점이 사라진다.
- 만들기: `qpdf --linearize in.pdf out.pdf`. 단, **선형화와 [[PDF 구조와 QDF|QDF]]는 동시에 못 쓴다**(선형화를 켜면 QDF가 자동 해제 — 객체를 사람이 읽기 쉽게 정렬하는 QDF와, 스트리밍용으로 특수 정렬하는 선형화가 상충하기 때문).

## 왜 중요한가
- 웹에서 큰 PDF(보고서·매뉴얼)를 **즉시 열리게** 하는 표준 최적화 — 체감 로딩이 크게 빨라진다.
- "왜 어떤 PDF는 바로 뜨고 어떤 건 다 받아야 뜨나"의 답이자, PDF 내부 배치(물리적 순서)가 **용도에 따라 달라진다**는 좋은 예.

## 관련 개념
