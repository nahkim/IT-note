# JSON & 직렬화 (Serialization)

## 한 줄 정의
**JSON**은 데이터를 주고받는 가장 흔한 텍스트 형식, **직렬화**는 메모리 속 객체를 그런 전송·저장 가능한 형태로 바꾸는 것.

## 자세히
- **JSON(JavaScript Object Notation)**:
  - **키-값 쌍**(`{ "name": "kim", "age": 30 }`)과 배열로 데이터를 표현하는 가볍고 사람이 읽기 쉬운 텍스트 형식.
  - JavaScript에서 유래했지만 지금은 **언어 독립적** — 거의 모든 언어가 다룬다.
  - [[REST API]] 응답의 사실상 표준 형식.
- 비슷한 형식과 비교:
  - **XML** — 태그 기반 마크업. 무겁지만 표현력↑(예전 표준).
  - **YAML** — 들여쓰기 기반. 사람이 읽기 가장 편해 **설정 파일**(쿠버네티스·CI)에 자주 쓰임.
- **직렬화(Serialization) / 역직렬화(Deserialization)**:
  - **직렬화** = 메모리의 객체 → 전송·저장 가능한 형태(JSON 문자열, 바이트). (예: JS `JSON.stringify`)
  - **역직렬화** = 그 형태 → 다시 객체. (예: `JSON.parse`)
  - 네트워크로 데이터를 보내거나 파일·DB에 저장하려면 반드시 거치는 과정.

## 왜 중요한가
- API·설정·로그·DB 등 데이터가 오가는 모든 곳에서 매일 만나는 형식.
- "객체를 어떻게 네트워크로 보내지?"의 답이 직렬화 — 백엔드의 기본 개념.
