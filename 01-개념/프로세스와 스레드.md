# 프로세스 vs 스레드 (Process / Thread)

## 한 줄 정의
**프로세스**는 실행 중인 프로그램의 독립된 단위(각자 메모리를 가짐), **스레드**는 한 프로세스 안에서 메모리를 공유하며 도는 작업 흐름.

## 자세히
- **프로세스(Process)**
  - 실행 중인 프로그램 하나. **자체 메모리 공간**(코드·데이터·힙·스택)과 자원을 가진다.
  - 서로 **격리(isolation)**되어 한 프로세스가 죽어도 다른 프로세스는 안전.
  - 프로세스 간 데이터 교환은 별도 통신(IPC)이 필요해 무겁다.
- **스레드(Thread)**
  - 한 프로세스 안의 실행 흐름. **각자 스택은 따로** 갖되, **코드·데이터·힙(메모리)은 프로세스 안의 다른 스레드와 공유**한다.
  - 공유 덕에 통신이 빠르고 가볍다. 대신 한 스레드가 공유 데이터를 잘못 건드리면 다른 스레드까지 영향 → **[[동시성 문제|동시성 버그]]**(레이스 컨디션, 교착상태) 위험.
- 비유: 프로세스 = 각자 작업실, 스레드 = 한 작업실 안에서 같은 책상을 나눠 쓰는 여러 일꾼.

## 왜 중요한가
- 멀티스레딩·동시성 프로그래밍의 출발점. "왜 락(lock)이 필요한가"가 여기서 나온다.
- 서버가 동시에 여러 요청을 처리하는 방식([[동기와 비동기]], 워커 모델)을 이해하는 토대.

## 관련 개념
- [[동기와 비동기]] — 한 스레드로도 여러 작업을 다루는 비동기 모델
- [[메시지 큐]] — 무거운 작업을 별도 워커(프로세스)로 넘기는 방식
- [[수평 확장]] — 프로세스/서버 단위로 늘려 부하 분산
