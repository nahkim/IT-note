# DNS (Domain Name System, 도메인 네임 시스템)

## 한 줄 정의
사람이 읽는 **도메인 이름(example.com)**을 컴퓨터가 쓰는 **IP 주소(93.184.x.x)**로 바꿔주는 인터넷의 전화번호부.

## 자세히
- 브라우저는 도메인 이름만으로는 접속할 수 없다. 먼저 DNS로 **이름 → IP** 변환(이름 해석, resolution)이 필요하다.
- 해석 과정(단순화):
  1. OS·브라우저 **캐시** 확인 → 있으면 끝.
  2. 없으면 **재귀 리졸버(recursive resolver)**(ISP나 1.1.1.1·8.8.8.8 같은 공용 DNS)에게 묻는다.
  3. 리졸버가 대신 **루트 → TLD(.com) → 권한(authoritative) 네임서버** 순으로 물어 최종 IP를 찾아온다.
- 주요 **레코드 타입**:
  - **A** — 도메인 → IPv4 주소.
  - **AAAA** — 도메인 → IPv6 주소.
  - **CNAME** — 도메인을 **다른 도메인의 별칭**으로 연결.
  - **MX** — 그 도메인의 **메일**을 받을 서버 지정.
- **캐싱**: 각 단계마다 결과를 일정 시간(TTL) 저장해, 매번 전체 과정을 반복하지 않아 빠르다.

## 왜 중요한가
- 모든 웹 접속의 첫 단계. 배포·도메인 연결·이메일 설정에서 반드시 만지게 된다.
- 장애·전파 지연(propagation)·캐시 문제는 실무 트러블슈팅의 단골 주제다.

## 관련 개념
- [[TCP-IP|TCP/IP]] — DNS가 알려주는 IP 주소로 실제 연결이 이뤄진다
- [[CDN]] — CDN도 DNS로 사용자를 가까운 엣지로 보낸다
- [[HTTP와 HTTPS]] — 이름 해석 다음 단계가 HTTP(S) 요청
