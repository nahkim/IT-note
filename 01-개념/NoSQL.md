# NoSQL (Not Only SQL)

## 한 줄 정의
표(table)·고정 스키마에 얽매이지 않는 **비관계형** 데이터베이스들의 통칭. 유연한 구조와 [[수평 확장]]에 강하다.

## 자세히
- "관계형([[관계형 데이터베이스와 SQL|RDB]])이 안 맞는 경우"를 위해 등장. 정형이 아닌 데이터, 막대한 규모, 빠른 확장이 필요할 때.
- 4가지 대표 유형:
  - **문서(Document)** — JSON 같은 문서 단위로 저장. 중첩 구조 가능. (예: **MongoDB**)
  - **키-값(Key-Value)** — `키 → 값`의 단순·초고속 저장소. (예: **Redis**, DynamoDB)
  - **와이드 칼럼(Wide-Column)** — 행마다 컬럼이 달라도 되는 열 지향. 대규모에 강함. (예: **Cassandra**, HBase)
  - **그래프(Graph)** — 노드(개체)와 엣지(관계)로 저장. 관계 탐색에 강함. (예: **Neo4j**)
- 일반적 경향: RDB는 **수직 확장**에 가깝고 정합성↑, NoSQL은 **[[수평 확장]]**에 강하고 유연성↑(대신 조인·강한 일관성은 약할 수 있음 → [[CAP 정리]]).
- "SQL vs NoSQL"은 우열이 아니라 **용도 선택**. 한 시스템에 둘을 함께 쓰는 경우도 많다.

## 왜 중요한가
- 대규모·실시간·비정형 데이터 서비스에서 흔한 선택지. RDB와의 트레이드오프를 이해해야 올바로 고른다.
- [[캐시|Redis]]·[[벡터DB]]도 넓게 보면 NoSQL 계열의 사고방식과 닿아 있다.
