# 프롬프트 엔지니어링 (Prompt Engineering)

## 한 줄 정의
원하는 답을 얻기 위해 [[LLM]]에게 주는 **입력(프롬프트)을 설계·다듬는** 기술.

## 자세히
- 기본기: **명확한 지시 + 충분한 맥락 + 예시 + 출력 형식 지정**.
- 대표 기법:
  - **Few-shot** — 예시 몇 개를 함께 제공.
  - **Chain-of-Thought** — "단계적으로 생각해봐"처럼 추론 과정을 유도.
  - **역할 지정(system 프롬프트)** — 모델의 페르소나·규칙을 고정.
  - **출력 형식 강제** — JSON 등 정해진 구조로 답하게 함.
- [[RAG]]는 검색한 문서를 프롬프트에 끼워 넣는 일종의 프롬프트 구성 기법이기도 하다.
- 모델이 똑똑해질수록 잔기술은 줄지만, **맥락·지시 설계**의 중요성은 그대로 남는다.

## 왜 중요한가
- 같은 모델도 프롬프트에 따라 결과 품질이 크게 달라진다 → 가장 싸고 빠른 성능 개선 수단.
- [[AI 에이전트]]의 행동도 결국 프롬프트(지시·도구 설명)로 제어된다.

## 관련 개념
- [[LLM]] — 프롬프트를 받는 대상
- [[RAG]] — 검색 결과를 프롬프트로 조립
- [[AI 에이전트]] — 프롬프트로 행동을 지시
