# GPU vs CPU

## 한 줄 정의
**CPU**는 소수의 강력한 코어로 **복잡한 일을 빠르게(저지연)**, **GPU**는 수천 개의 단순 코어로 **같은 연산을 대량 동시에(고처리량)** 처리한다. "적은 일을 다양하고 빠르게" vs "**같은 일을 엄청 많이**".

## 자세히

### 설계 철학의 차이
- **CPU — 지연(latency) 최적화** — 큰 캐시·분기 예측·비순차 실행(OOO)으로 **단일 작업을 최대한 빨리**. 트랜지스터를 '똑똑한 제어'에 많이 쓴다.
- **GPU — 처리량(throughput) 최적화** — 제어 로직을 줄이고 **연산 유닛(코어)을 최대한 많이** 박아 **대량 병렬**. 메모리 지연은 '수많은 스레드로 가려서(oversubscription)' 견딘다.

### 비교표
| 기준 | CPU | GPU |
|---|---|---|
| 코어 | 소수(수~수십), 강력 | **수천**, 단순 |
| 최적화 | **지연**(빨리 끝) | **처리량**(많이 처리) |
| 실행 모델 | 복잡한 제어·OOO·분기예측 | **SIMT/SIMD**(같은 명령을 여러 데이터에) |
| 메모리 | 대용량 RAM | **초고대역폭 VRAM(HBM)** |
| 잘하는 일 | 순차·분기·범용 | 데이터 병렬(대량 동일 연산) |

### 각각 빠르게 처리하는 일
- **CPU가 빠른 일** — 분기 많은 로직, **순차 의존** 작업, 단일 스레드 성능, OS·앱 제어, DB 쿼리, I/O·네트워크 조율, 다양한 범용 태스크, 낮은 지연 응답.
- **GPU가 빠른 일** — **행렬곱·컨볼루션**(딥러닝 [[Transformer]]), 그래픽 렌더링, 이미지/영상 처리, 대규모 병렬 수치연산(과학·시뮬레이션), 같은 연산을 **수백만 데이터에** 한꺼번에.

### 왜 AI는 GPU인가
- 딥러닝의 핵심은 거대한 **행렬곱/벡터 연산** → 데이터 병렬성이 극대 → GPU 수천 코어가 압도적. 최신 GPU(H100 등)는 **Tensor Core**(행렬 곱-누적 전용)와 **HBM**으로 초당 테라바이트급 처리. [[LLM]] 학습·추론이 GPU에 의존([[CUDA]]·[[vLLM]]).

### 함께 일한다 (호스트-디바이스)
- 실제론 **CPU가 전체를 조율(호스트)**, 무거운 병렬 부분을 **GPU(디바이스)**에 위임한다.
- CPU↔GPU **데이터 전송 오버헤드**가 있어, 병렬화 이득이 그걸 넘어야 GPU가 유리. **순차 부분이 많으면 GPU 이득이 제한**된다(암달의 법칙).
- 더 좁게 특화한 **TPU/NPU**는 AI 연산 전용 가속기(GPU보다 범용성↓·효율↑).

## 왜 중요한가
- "이 작업을 CPU로? GPU로?"는 성능·비용의 근본 선택 — **병렬화 가능한 대량 동일 연산이면 GPU, 분기·순차·범용이면 CPU**.
- AI 붐 = **GPU 수요 폭발**의 근본 이유(대규모 행렬 연산 = GPU에 최적).
