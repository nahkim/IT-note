# API 게이트웨이 (API Gateway)

## 한 줄 정의
여러 백엔드 서비스 앞에 두고 **모든 API 요청을 한 곳에서 받아 라우팅·공통 처리**하는 관문.

## 자세히
- 클라이언트는 수많은 서비스 주소를 알 필요 없이 **게이트웨이 한 곳**에만 요청한다.
- 공통 기능을 한데 모은다: **인증·인가, 요청 라우팅, 속도 제한(rate limit), 로깅, 응답 합치기**.
- [[리버스 프록시]]와 겹치지만, 게이트웨이는 더 **API·서비스 지향**이다(인증·요금제·버전 관리 같은 비즈니스 기능까지 포함).
- 대표: Kong, AWS API Gateway, Nginx 기반 게이트웨이.

## 왜 중요한가
- [[마이크로서비스]]에서 클라이언트와 내부 서비스 사이의 **단일 진입점** 역할.
- 인증·제한 같은 공통 관심사를 각 서비스가 중복 구현하지 않게 모아준다.
