# 시간 복잡도 / Big-O (Time Complexity)

## 한 줄 정의
입력 크기 n이 커질 때 **알고리즘의 실행 시간이 얼마나 빨리 늘어나는지**를 나타내는 표기법(보통 최악의 경우).

## 자세히
- 실제 초 단위 시간이 아니라 **증가 추세(성장률)**를 본다. 상수·계수는 무시.
- 자주 나오는 복잡도(빠른 → 느린 순):
  - **O(1)** 상수 — 입력 크기와 무관. (배열 인덱스 접근, [[자료구조|해시 테이블]] 조회)
  - **O(log n)** 로그 — 매 단계 절반씩 줄임. (이진 탐색, [[데이터베이스 인덱스|B-tree]] 탐색)
  - **O(n)** 선형 — 입력에 비례. (배열 한 번 순회)
  - **O(n log n)** — 효율적인 정렬(병합·퀵 정렬)의 대표 복잡도.
  - **O(n²)** 제곱 — 이중 반복문. 입력 10배면 시간 100배 → 큰 데이터에 부적합. (버블 정렬)
- 직관: n=1,000,000일 때 O(n²)는 1조 번 연산이지만 O(log n)은 약 20번이면 끝.

## 왜 중요한가
- "왜 데이터가 커지니 느려지는가"를 설명하는 공용어. 자료구조·알고리즘 선택의 판단 기준.
- 면접·실무에서 "이 코드의 복잡도는?"은 기본 질문이다.
