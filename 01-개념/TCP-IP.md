# TCP/IP (그리고 TCP vs UDP)

## 한 줄 정의
인터넷 통신의 기반이 되는 프로토콜 묶음. **IP**가 데이터를 주소로 배달하고, 그 위에서 **TCP/UDP**가 실제 데이터를 주고받는 방식을 정한다.

## 자세히
- **IP (Internet Protocol)**: 데이터를 작은 **패킷**으로 쪼개, **IP 주소**를 보고 목적지까지 라우팅한다. 단, 도착·순서를 보장하진 않는다.
- 그 위 **전송 계층**에 TCP와 UDP가 있다.

| | **TCP** | **UDP** |
|---|---|---|
| 연결 | 연결 지향(connection-oriented) | 비연결(connectionless) |
| 신뢰성 | 도착·순서 **보장**(재전송·확인응답) | 보장 안 함(빠르지만 유실 가능) |
| 속도 | 상대적으로 느림 | 빠름·가벼움 |
| 용도 | 웹·이메일·파일 등 정확성이 중요한 것 | 영상 스트리밍·게임·실시간 통화·DNS |

- **TCP 3-way handshake**: 연결을 열 때 **SYN → SYN-ACK → ACK** 3단계로 양쪽이 준비됐는지 확인한 뒤 데이터를 보낸다.
- [[HTTP와 HTTPS|HTTP]]는 보통 이 TCP 위에서 돈다(즉, 신뢰성 있는 연결 위의 요청-응답).

## 왜 중요한가
- 모든 네트워크 통신의 바닥. "왜 연결이 느리지/끊기지"를 이해하려면 TCP/IP를 알아야 한다.
- TCP vs UDP 선택은 "**정확성 vs 속도**"라는 전형적 트레이드오프 — 서비스 성격에 맞춰 고른다.

## 관련 개념
- [[DNS]] — 도메인을 IP로 바꾸면, 그 IP로 TCP/IP가 연결한다
- [[HTTP와 HTTPS]] — TCP 위에서 도는 응용 계층 프로토콜
- [[로드 밸런서]] — L4(TCP/IP) vs L7(HTTP) 계층 구분의 출발점
