# Git / 버전 관리 (Version Control)

## 한 줄 정의
코드의 **변경 이력을 기록·되돌리고 여러 사람이 함께 작업**하게 해주는 도구. Git은 그중 가장 널리 쓰이는 분산 버전 관리 시스템.

## 자세히
- **분산(distributed)**: 모든 개발자가 **전체 히스토리의 완전한 복사본**을 자기 컴퓨터에 갖는다 → 오프라인 작업·빠른 속도·안정성.
- 핵심 개념:
  - **커밋(commit)** — 그 시점의 파일 상태를 스냅샷으로 기록(이전 커밋과 연결).
  - **브랜치(branch)** — 커밋들의 흐름을 가리키는 포인터. 새 기능·수정을 **격리된 줄기**에서 작업.
  - **머지(merge)** — 브랜치를 다시 합침. 같은 부분을 둘 다 고쳤으면 **충돌(conflict)**을 직접 해결해야 함.
- **Git ≠ GitHub**:
  - **Git** = 내 컴퓨터에서 도는 *버전 관리 도구* 자체.
  - **GitHub/GitLab** = Git 저장소를 온라인에 **호스팅**하고 코드 리뷰·이슈·[[CI-CD|자동화]]를 더해주는 *서비스*.

## 왜 중요한가
- 모든 협업 개발의 기본 인프라. 커밋·브랜치·PR(풀 리퀘스트) 없이는 팀 개발이 불가능하다.
- 실수해도 과거로 되돌릴 수 있다는 안전망 — 주니어가 가장 먼저, 매일 쓰는 도구.
