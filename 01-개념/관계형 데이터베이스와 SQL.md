# 관계형 데이터베이스 & SQL (RDB / SQL)

## 한 줄 정의
데이터를 **표(table)** 형태로 저장하고, 표 사이의 **관계**를 키로 연결하는 데이터베이스. 그 표를 다루는 언어가 **SQL**.

## 자세히
- 구조: **테이블 = 행(row, 레코드) × 열(column, 속성)**. 미리 정한 **스키마(컬럼·타입)**에 맞춰 정형 데이터를 저장한다.
- **키(key)**로 관계를 만든다:
  - **기본키(Primary Key)** — 각 행을 유일하게 식별(예: `user_id`).
  - **외래키(Foreign Key)** — 다른 테이블의 기본키를 참조해 표끼리 연결(예: 주문 테이블의 `user_id`).
- **SQL(Structured Query Language)**: 표를 조회·조작하는 표준 언어.
  - `SELECT`(조회) `INSERT`(추가) `UPDATE`(수정) `DELETE`(삭제), 여러 표를 합치는 `JOIN`.
- 강점: **데이터 정합성**(제약·외래키), 복잡한 질의/조인, [[트랜잭션과 ACID|트랜잭션(ACID)]].
- 대표 제품: PostgreSQL, MySQL, MariaDB, SQLite, Oracle, SQL Server.

## 왜 중요한가
- 가장 보편적인 데이터 저장 방식. 백엔드 개발에서 거의 항상 마주친다.
- 정합성이 중요한 도메인(금융·주문·회원)의 기본 선택지. 비교 대상은 [[NoSQL]].
