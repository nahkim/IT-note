# CUDA

## 한 줄 정의
NVIDIA가 만든 **병렬 컴퓨팅 플랫폼 + 프로그래밍 모델**. GPU의 수천 개 코어를 그래픽 외 **범용 계산(GPGPU)**에 쓸 수 있게 해, 딥러닝·과학연산처럼 '같은 연산을 대량으로' 하는 작업을 CPU보다 수십~수백 배 빠르게 돌린다. (2007년 공개)

## 자세히

### 왜 GPU인가 (CPU와의 차이)
- **CPU** — 소수의 강력한 코어. 복잡한 분기·순차 로직에 강함(저지연 중심).
- **GPU** — 수천 개의 단순한 코어. **같은 연산을 대량의 데이터에 동시에**(고처리량 중심). 행렬곱·벡터 연산처럼 '같은 일을 엄청 많이' 하는 작업에 최적.
- **CUDA**는 이 GPU를 C/C++/Fortran·**Python** 등에서 직접 프로그래밍하게 해주는 플랫폼. NVIDIA GPU 전용이라는 점이 핵심(범용 표준인 OpenCL과 대비).

### 핵심 개념
- **호스트(host) · 디바이스(device)** — CPU·메인메모리 = 호스트, GPU·GPU메모리 = 디바이스. 데이터를 **호스트 ↔ 디바이스로 복사**해 가며 계산한다(이 복사 비용이 성능의 큰 변수).
- **커널(kernel)** — GPU에서 **수천 스레드가 동시에 실행**하는 함수(`__global__`). `kernel<<<블록 수, 블록당 스레드 수>>>(...)` 형태로 호출.
- **스레드 계층** — 스레드(thread) → **블록(block)** → **그리드(grid)**. 한 블록은 하나의 **SM(Streaming Multiprocessor)**에서 실행되고, 같은 블록의 스레드끼리는 **공유 메모리**로 빠르게 협력한다.
- **워프(warp)와 SIMT** — GPU는 스레드를 **32개 단위(warp)**로 묶어 한 명령을 동시에 처리한다(**SIMT**, Single-Instruction Multiple-Thread). 그래서 워프 안에서 `if`로 길이 갈리면(warp divergence) 성능이 떨어진다.
- **메모리 계층** — 레지스터 > 공유 메모리(빠름·블록 내 공유) > 글로벌 메모리(큼·느림) > 상수/텍스처. **이 계층을 얼마나 잘 쓰느냐가 최적화의 핵심.** (GC가 알아서 해주는 [[가비지 컬렉션과 메모리]]와 달리 **수동 관리**)
- **스트림(stream)** — 연산과 메모리 복사를 **비동기로 겹쳐(overlap)** 파이프라인처럼 돌려 처리량을 높인다.

### 생태계 (CUDA가 강한 '진짜' 이유)
- **컴파일러·툴킷** — `nvcc` 컴파일러, **CUDA Toolkit**(2026-06 현재 **13.3**), `nvidia-smi`, Nsight 프로파일러.
- **고수준 라이브러리** — **cuBLAS**(선형대수)·**cuDNN**(딥러닝 연산)·**NCCL**(다중 GPU·노드 간 통신)·Thrust·cuFFT 등. PyTorch·TensorFlow가 내부에서 이들을 호출한다.
- **아키텍처 / Compute Capability** — GPU 세대마다 기능 버전이 다름: Turing → Ampere → Ada → Hopper(H100/H200) → **Blackwell**(최신, FP4 등 저정밀 연산 지원).

### 어디에 쓰이나
- **딥러닝 / LLM** — [[Transformer]] 학습·추론의 거대한 행렬 연산이 전부 GPU에서 돈다. PyTorch에서 `tensor.to("cuda")`가 바로 이것.
- HPC·과학 시뮬레이션, 영상/렌더링, 대규모 데이터 분석(RAPIDS) 등.

## 왜 중요한가
- **AI 붐의 소프트웨어 토대** — 사실상 모든 주요 딥러닝 프레임워크가 CUDA 위에서 동작한다. [[LLM]] 학습·서빙 = CUDA라고 봐도 무방.
- **NVIDIA의 해자(moat)** — 15년 넘게 쌓인 라이브러리·문서·커뮤니티가 강력한 **락인(lock-in)**. 경쟁자(AMD **ROCm**, **OpenCL**, Intel oneAPI, Apple Metal)가 하드웨어는 따라와도 **소프트웨어 성숙도** 격차가 남아 있다(2026년 벤치마크상 동일 사양에서 CUDA가 OpenCL 대비 약 30~50% 우위). ROCm이 빠르게 추격 중이나 여전히 CUDA가 표준.
- **확장성** — 다중 GPU·분산 학습([[수평 확장]])도 NCCL 등 CUDA 생태계로 묶여 돌아간다.
