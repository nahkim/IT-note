# 데이터베이스 샤딩 (Database Sharding)

## 한 줄 정의
하나의 큰 DB를 **여러 조각(샤드)으로 쪼개** 여러 서버에 나눠 저장하는 것.

## 자세히
- DB가 한 대로 감당 안 될 때, 데이터를 기준(샤드 키)에 따라 분할한다.
- 예: 사용자 ID로 나누기 → ID 0~999만은 샤드 A, 1000~1999는 샤드 B …
- **샤드 키 선택**이 핵심 — 잘못 고르면 특정 샤드에만 부하가 몰림(핫스팟).
- 비슷한 개념: **파티셔닝**(한 DB 안에서 분할), **[[데이터베이스 복제]]**(레플리카 — 읽기 분산·장애 대비).
- 대가: 여러 샤드에 걸친 조인·트랜잭션이 어려워진다.

## 왜 중요한가
- 데이터 계층의 [[수평 확장]] 수단. 데이터가 폭발적으로 커지는 서비스의 필수 기법.
