# CAP 정리 (CAP Theorem)

## 한 줄 정의
분산 시스템은 **일관성(C)·가용성(A)·분단내성(P)** 중 셋을 동시에 완벽히 가질 수 없고, 보통 둘만 강하게 택한다.

## 자세히
- **C (Consistency, 일관성)**: 모든 노드가 항상 같은 최신 데이터를 보여준다.
- **A (Availability, 가용성)**: 일부 노드가 죽어도 요청에 항상 응답한다.
- **P (Partition tolerance, 분단내성)**: 노드 간 통신이 끊겨도 시스템이 동작한다.
- 현실의 분산 시스템에서 네트워크 분단(P)은 **피할 수 없으므로**, 실질적으로는 분단이 생겼을 때 **C냐 A냐**를 고르는 문제가 된다.
  - **CP 선택**: 일관성 우선 (예: 은행 잔액) — 분단 시 일부 응답을 막아서라도 데이터 정확성 유지.
  - **AP 선택**: 가용성 우선 (예: SNS 좋아요 수) — 잠깐 데이터가 어긋나도 계속 응답.

## 왜 중요한가
- 데이터를 여러 서버에 나누는 순간([[데이터베이스 샤딩]], [[수평 확장]]) 반드시 부딪히는 근본 트레이드오프.
- "어느 쪽이 틀려도 괜찮은가?"를 정하는 설계 판단의 기준.
