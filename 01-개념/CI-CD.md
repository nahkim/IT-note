# CI/CD (지속적 통합 / 지속적 배포)

## 한 줄 정의
코드를 합치고(빌드·테스트), 내보내는(배포) 과정을 **자동화한 파이프라인**. 수동 작업을 줄여 더 자주·안전하게 릴리스한다.

## 자세히
- **CI (Continuous Integration, 지속적 통합)**
  - 개발자들이 코드를 **자주 중앙 저장소에 병합**하고, 그때마다 **자동 빌드 + 자동 테스트**가 돈다.
  - 효과: 통합 충돌·버그를 일찍 발견. ("merge 했더니 다 깨졌다"를 방지)
- **CD — 두 의미가 있다**:
  - **Continuous Delivery(지속적 전달)** — 언제든 배포 가능한 상태까지 자동화하되, **운영 반영은 사람이 승인**.
  - **Continuous Deployment(지속적 배포)** — 테스트를 통과하면 **사람 승인 없이 자동으로 운영까지** 배포.
- 흐름(파이프라인): `코드 푸시 → 빌드 → 자동 테스트 → (스테이징) → 배포`. [[Git]] 푸시가 보통 방아쇠.
- 대표 도구: GitHub Actions, GitLab CI, Jenkins, CircleCI.

## 왜 중요한가
- 배포를 "큰 행사"에서 "일상"으로 바꾼다 — 작게 자주 내보내 위험을 줄이는 현대 개발의 핵심.
- 테스트 자동화와 결합해 품질을 지키면서 속도를 낸다.

## 관련 개념

- [[컨테이너와 Docker]] — 빌드 결과물을 이미지로 만들어 배포
- [[쿠버네티스]] — 배포 대상 인프라
