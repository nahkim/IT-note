# 리버스 프록시 (Reverse Proxy)

## 한 줄 정의
클라이언트와 서버 사이에서 **서버를 대신해 요청을 받아주는** 중간 관문.

## 자세히
- 클라이언트는 진짜 서버가 아니라 **리버스 프록시에게 요청**하고, 프록시가 뒤의 서버로 전달한다.
- (참고) **포워드 프록시**는 *클라이언트*를 대신하지만, 리버스 프록시는 *서버*를 대신한다 — 방향이 반대.
- 주요 역할:
  - **요청 분배** → [[로드 밸런서]] 기능
  - **[[캐시]]** → 응답을 저장해 재사용
  - **SSL 종료** → HTTPS 암복호화를 대신 처리
  - **보안** → 실제 서버 주소를 감추고, 공격을 앞단에서 차단
- 대표: Nginx, HAProxy, Envoy.

## 왜 중요한가
- 서버 구조를 숨기고, 인증·캐시·압축·로드밸런싱 같은 공통 기능을 **한 곳에 모아** 처리하게 해준다.

## 관련 개념
- [[로드 밸런서]] — 리버스 프록시가 흔히 겸하는 역할
- [[CDN]] — 지리적으로 분산된 리버스 프록시의 일종
- [[캐시]] — 리버스 프록시의 핵심 기능 중 하나
