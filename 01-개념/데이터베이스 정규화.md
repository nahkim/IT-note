# 데이터베이스 정규화 (Normalization)

## 한 줄 정의
데이터 **중복을 줄이고 정합성을 지키도록** 테이블을 쪼개 정리하는 설계 원칙.

## 자세히
- 한 테이블에 모든 걸 욱여넣으면 **중복 저장 → 갱신 누락·불일치**(이상 현상, anomaly)가 생긴다. 정규화는 이를 규칙적으로 분리한다.
- **정규형(Normal Form)**은 누적적이다 — 2NF는 1NF를 포함, 3NF는 2NF를 포함.
  - **1NF(제1정규형)**: 한 칸에는 **하나의 값만**(원자값). 반복 그룹 제거.
  - **2NF(제2정규형)**: 1NF + 모든 비(非)키 컬럼이 **기본키 전체에** 의존(부분 종속 제거).
  - **3NF(제3정규형)**: 2NF + 비키 컬럼끼리 서로 의존하지 않음(**이행 종속** 제거).
- 보통 **3NF까지**를 실무 기준으로 본다.
- 반대 방향 **비정규화(denormalization)**: 일부러 중복을 둬서 조인을 줄이고 **읽기 속도**를 높이는 선택 — 성능을 위해 정합성 일부를 양보.

## 왜 중요한가
- DB 테이블 설계의 기본 사고법. 잘못된 스키마는 두고두고 버그와 느린 쿼리를 부른다.
- "정규화(정합성) vs 비정규화(읽기 성능)"는 [[캐시]]·[[데이터베이스 인덱스|인덱스]]와 같은 결의 트레이드오프 판단이다.
