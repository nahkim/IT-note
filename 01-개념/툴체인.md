# 툴체인 (Toolchain)

## 한 줄 정의
소프트웨어를 빌드·개발하는 데 쓰는 **도구들의 묶음**. 한 도구의 출력이 다음 도구의 입력이 되도록 **사슬(chain)처럼 연결**돼 있다(예: 컴파일러 → 링커 → 디버거).

## 자세히

### 구성 요소
- **컴파일러** — 소스 코드를 오브젝트 코드로 변환([[컴파일러와 인터프리터]]).
- **어셈블러** — 어셈블리를 기계어로.
- **링커(linker)** — 여러 오브젝트 파일·라이브러리를 하나의 실행 파일로 묶음.
- **런타임 라이브러리** — OS·외부 기능과 연결.
- **디버거 / 프로파일러** — 실행 중 문제 진단.

### 툴체인 vs 빌드 시스템 vs SDK vs IDE (헷갈리기 쉬움)
- **툴체인** — 변환 도구들의 사슬 자체(컴파일러/링커…).
- **빌드 시스템**(Make·CMake·Cargo·Gradle) — 툴체인을 **언제·어떤 순서로 부를지 자동화**(오케스트레이션).
- **SDK** — 툴체인 + 라이브러리 + 문서 + 에뮬레이터 등을 묶은 '개발 키트'(더 넓은 개념).
- **IDE** — 이 모든 걸 감싼 편집·실행 환경.

### 크로스 컴파일 툴체인
빌드하는 머신(host)과 실행 대상(target)이 **다를 때** 쓰는 툴체인. 예: x86 PC에서 ARM 임베디드·모바일용 바이너리를 뽑을 때. → [[온디바이스]]·임베디드 개발의 핵심.

### 예시
- **Rust**: `rustc`(컴파일러) + `cargo`(빌드/패키지) + `rustup`(툴체인 버전 관리) → [[Tauri]]가 이 위에서 돈다.
- **LLVM/Clang**, **GCC + binutils**, Android **SDK/NDK**, [[CUDA]]의 `nvcc`.

## 왜 중요한가
- "빌드가 어떻게 되는가"의 뼈대. **툴체인 버전이 결과물·재현성을 좌우**한다(같은 소스도 툴체인이 다르면 산출물이 달라질 수 있음).
- 크로스플랫폼([[Tauri]] 등)·임베디드·[[온디바이스]]에서 특히 중요 — 대상 아키텍처에 맞는 툴체인이 있어야 빌드가 된다.
