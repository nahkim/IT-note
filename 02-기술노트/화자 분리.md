# 📝 화자 분리 (Speaker Diarization)

## 한 줄 요약
오디오를 **"누가, 언제 말했나"** 로 쪼개는 기술 — "Speaker 1: 0~5초, Speaker 2: 5~9초". 미리 누군지 아는 게 아니라(그건 화자 *식별*) **익명의 화자들을 구분만** 한다. [[STT 음성 인식]]과 합치면 화자별 회의록이 된다.

## 핵심 내용

### 1. 화자 분리 ≠ 화자 식별 ≠ STT
- **화자 분리(Diarization)** — "Speaker A/B/C"로 **구분**. 누군지 이름은 모른다.
- **화자 식별/검증(Speaker ID/Verification)** — 등록된 사람과 대조해 "이건 김씨다"라고 **특정**.
- **STT** — 말의 *내용*을 텍스트로. (셋은 별개 문제이고 보통 조합해서 쓴다)

### 2. 전통적 파이프라인 (모듈 결합형)
순서대로 쌓는 고전 방식:
1. **VAD (Voice Activity Detection)** — 말소리 구간과 침묵/잡음을 가른다. (임베딩 추출기는 침묵을 잘 표현 못하므로 먼저 걸러야 함)
2. **분할(Segmentation)** — 말소리 구간을 짧은 조각으로 쪼갠다.
3. **화자 임베딩 추출** — 각 조각을 화자 특징 벡터로. **i-vector → d-vector → x-vector → ECAPA-TDNN**으로 발전. ([[임베딩]] — 같은 사람 목소리는 벡터 공간에서 가깝게 모인다)
4. **클러스터링** — 비슷한 임베딩끼리 묶어 화자 수만큼 그룹화. 응집형 계층 클러스터링(AHC)·**스펙트럴 클러스터링**·VBx(베이지안 HMM) 등.
- **약점**: 단계마다 오류가 쌓이고, **겹쳐 말하기(overlap)**를 제대로 못 다루며, DER을 직접 최소화하도록 학습되지 않는다.

### 3. End-to-End 신경망 방식 (EEND)
- 위 단계를 쪼개지 않고 **신경망 하나가** 멀티화자 오디오 → 프레임별 화자 활성을 **바로** 출력.
- 화자 분리를 **다중 레이블 분류** 문제로 보고, 화자 순서에 무관한 **순열 불변 학습(PIT)**으로 DER을 직접 최적화.
- **장점**: **겹쳐 말하기를 명시적으로 처리**, 단계 누적 오류 없음. BLSTM·[[Transformer]]·Conformer 구조 사용.
- 화자 수 가변·온라인(스트리밍) 확장형(attractor 기반)도 활발.

### 4. 핵심 지표: DER (Diarization Error Rate)
- **DER = (놓친 말소리 + 잘못 잡은 말소리 + 화자 혼동) / 전체 시간.** 낮을수록 좋다.
- 특히 **겹쳐 말하기**와 **잡음 환경**에서 점수가 크게 갈린다 → 최근 발전의 주 전장.

### 5. 대표 도구·모델 (2025~2026)
- **pyannote.audio** — 오픈소스의 **사실상 표준**. VAD+임베딩+클러스터링 파이프라인을 잘 묶어 제공, 적당한 하드웨어로 범용적으로 무난. 연구·커스터마이즈에 강점.
- **NVIDIA NeMo — Sortformer** — 18층 Transformer 기반 **end-to-end** 방식. 파이프라인 단계를 없애 단일 문제로. 성능 좋지만 **GPU 자원을 많이** 먹는다. (이전 세대: MSDD)
- **WhisperX** — Whisper 전사 + (pyannote) 화자 분리 + 강제 정렬을 묶어 **화자별 타임스탬프 자막**을 한 번에. 실무에서 인기.
- **Diart** — **실시간(스트리밍)** 화자 분리에 특화된 오픈소스.
- **상용 API** — AssemblyAI·Deepgram 등이 STT에 화자 분리를 옵션으로 제공. (AssemblyAI는 2025년 새 화자 임베딩 모델로 잡음 환경 ~30% 개선 발표)

### 6. 무엇을 고를까
- **범용·오픈소스·커스터마이즈** → pyannote.audio.
- **겹쳐 말하기 많고 GPU 충분** → NeMo Sortformer(EEND 계열).
- **화자별 자막/회의록을 빠르게** → WhisperX([[STT 음성 인식]]+분리 통합).
- **실시간** → Diart, 또는 스트리밍 지원 상용 API.
- **운영 부담 없이 API로** → AssemblyAI / Deepgram.

## 코드 / 예시
```
[오디오]
   │
   ▼ (전통적 파이프라인)
[VAD] → [분할] → [화자 임베딩(ECAPA-TDNN)] → [클러스터링] → "누가 언제"
   │
   ▼ (End-to-End / EEND·Sortformer)
[신경망 하나] ───────────────────────────────────→ "누가 언제" (겹침까지)
```
```python
# pyannote.audio — 화자 분리 한 번 돌리기
from pyannote.audio import Pipeline
pipe = Pipeline.from_pretrained("pyannote/speaker-diarization-3.1",
                                use_auth_token="hf_...")
diar = pipe("meeting.wav")
for turn, _, speaker in diar.itertracks(yield_label=True):
    print(f"{turn.start:.1f}s-{turn.end:.1f}s  {speaker}")
# → 0.0s-5.2s SPEAKER_00 / 5.2s-9.1s SPEAKER_01 ...
```

## 기억할 문장 / 핵심 포인트
> "화자 분리는 '누군지'가 아니라 **'몇 명을, 언제로 나눌지'** 다 — 이름은 모른 채 A·B·C로 가른다."
> "전통 파이프라인(VAD→임베딩→클러스터링)의 최대 약점은 **겹쳐 말하기**. EEND는 이걸 정면으로 다루려고 나왔다."
> "[[STT 음성 인식]]이 '무슨 말'이라면 화자 분리는 '누구 말' — 합쳐야 회의록이 완성된다(WhisperX)."
