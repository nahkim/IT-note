# 📝 PoC · 프로토타입 · MVP · 파일럿 — 검증 단계와 절차

## 한 줄 요약
아이디어를 제품으로 키우기 전, **무엇을 검증하느냐**에 따라 단계가 나뉜다: **PoC(기술 되나?) → 프로토타입(어떻게 생겼나?) → MVP(시장이 원하나?) → 파일럿(실환경에서 되나?) → 전면 도입**. 각 단계는 **사전 성공 기준 + go/no-go 게이트**를 갖는다.

## 핵심 내용

### 1. 각각의 의미 (무엇을 검증하나)
- **PoC (Proof of Concept)** — "**기술적으로 가능한가?**" 아이디어/기술의 **실현 가능성**을 소규모로 검증. 개발 착수 **전**, 내부, 대개 **버릴 코드**(샌드박스).
- **프로토타입(Prototype)** — "**어떻게 생기고 어떤 흐름인가?**" **형태·UX·디자인**을 보여주는 모형. 사용성·피드백 검증.
- **MVP (Minimum Viable Product)** — "**사람들이 실제로 쓰고/돈 낼까?**" **최소 기능 제품을 실제 출시**해 시장 수요·사용자 가치 검증.
- **파일럿(Pilot)** — "**실제 환경에서 운영되나?**" **제한된 실사용자·실데이터**로 시범 운영. 전면 확산 **직전의 마지막 관문**.
- (참고) **베타(Beta)** — 출시 전 넓은 사용자 대상 테스트([[릴리즈]] 채널). GA = 정식 출시.

### 2. 전형적 순서
```
아이디어 → PoC → 프로토타입 → (MVP) → 파일럿 → 전면 도입(Production/GA)
        기술성    형태·UX     시장성     실운영        확산
```
> 맥락 따라 순서·강조가 다르다 — **스타트업**은 MVP 중심(시장 검증), **기업 도입**은 PoC → 파일럿 중심(기술·운영 검증).

### 3. 한눈에 비교
| 단계 | 핵심 질문 | 검증 대상 | 규모·환경 | 산출물 |
|---|---|---|---|---|
| **PoC** | 기술적으로 되나? | 실현 가능성 | 소규모·내부·샌드박스 | 버릴 실험 코드 |
| **프로토타입** | 어떻게 생겼나? | 형태·UX | 목업/모형 | 디자인 모형 |
| **MVP** | 시장이 원하나? | 사용자·시장 가치 | **실제 출시**(최소기능) | 진짜 제품 |
| **파일럿** | 실환경에서 되나? | 실운영·확장성 | 제한된 실사용자·실데이터 | 운영 검증 결과 |

### 4. PoC 절차 (게이트 중심)
1. **문제·가설 + 정량 성공 기준 정의** — 정확도·[[레이턴시|지연]]·비용 등 **숫자로**(이게 없으면 대부분 실패).
2. **최소 구현** — 샌드박스에서 가장 빠르게.
3. **4대 리스크 병렬 검증** — ①기술 실현성 ②비즈니스 가치 ③**데이터 준비도** ④운영 준비도.
4. **go/no-go 게이트** — 결과는 항상 셋 중 하나: **Proceed(→파일럿) / Pivot(방향 수정) / Kill(중단)**.

### 5. 파일럿 절차
1. **대상 선정 + 파일럿 차터** — 시범 사용자/부서 + **성공 기준을 사전 합의**(나중에 골대 옮기는 것 방지).
2. **실제 운영 환경 배포** — 실데이터·실사용자.
3. **KPI·피드백 수집** — 완료율·시간 단축·만족도·장애.
4. **평가 → 확산(rollout) / 조정 / 중단** 결정. 확산은 [[무중단 배포]]로.

### 6. 함정 — 'PoC 무덤(PoC Purgatory)'
- **AI 프로젝트의 70~80%가 프로덕션에 못 간다**(일부 리포트는 95% 실패). 샌드박스에선 되는데 **비즈니스 가치·데이터 품질·조직 준비도**의 벽을 못 넘김.
- 원인: "AI가 도움되겠지" 정도로 시작해 **정량 가치·성공 기준을 안 정함** → 확산 시점에 "왜 하는지·성공이 뭔지" 합의가 안 됨.
- 처방: **착수 전에** 성공 기준·게이트를 못 박고, 기술+비즈니스+데이터를 **병렬로** 검증.

## 코드 / 예시
```
[PoC]  성공기준(정확도/지연/비용) 정의
   │  go/no-go ─▶ Kill(중단) / Pivot(수정)
   ▼ Proceed
[파일럿]  파일럿 차터 + 제한 실사용자·실데이터 + KPI
   │  평가 ─▶ 조정 / 중단
   ▼ 확산 결정
[전면 도입]  무중단 배포로 rollout
```

## 기억할 문장 / 핵심 포인트
> "**PoC는 '되나?', 파일럿은 '실제로 쓸만한가?'.**" — PoC는 기술 실현성(샌드박스), 파일럿은 실환경 운영 검증.
> **성공 기준을 시작 전에 숫자로 못 박아라** — 안 그러면 'PoC 무덤'에서 못 나온다.
> 모든 게이트의 결론은 셋뿐: **진행 · 수정 · 중단**.
