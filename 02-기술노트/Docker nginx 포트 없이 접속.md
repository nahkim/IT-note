# 📝 Docker + nginx — 포트 번호 없이 접속되는 이유

## 한 줄 요약
포트 없이 접속되는 건 **nginx '마법'이 아니라, 서비스가 기본 포트(HTTP 80 / HTTPS 443)에 떠 있기 때문**이다. nginx(리버스 프록시)는 80/443을 대신 점유하고 뒤의 앱 포트(예: 8082)로 넘겨줘서 포트를 감춰줄 뿐 — **도커에서 `-p 80:8082`만 해도** 포트 없이 접속된다. nginx의 진짜 가치는 **여러 앱 도메인 분기 + HTTPS**.

## 핵심 내용

### 1. 왜 포트가 "사라지나" — 기본 포트 규칙
- 브라우저는 URL에 포트가 없으면 **자동으로 `http://`→80, `https://`→443**에 접속한다.
- 즉 **"포트를 안 쓴다" = "그 서비스가 80/443에 떠 있다"**. 8082 같은 비기본 포트면 `http://host:8082`처럼 **써야** 한다. (→ [[OSI 7계층|L7]] 애플리케이션 계층 이야기)
- ⚠️ **이건 '리다이렉트'가 아니라 클라이언트가 알아서 붙이는 것**이다. 브라우저가 **접속 전에** 스킴 기본 포트(80/443)를 채워 바로 그 포트로 연결 — 서버 관여·네트워크 왕복·3xx 응답 **없음**(URL 스킴 기본 포트는 RFC 3986/IANA로 정의). 주소창은 `:80`/`:443`을 쳐도 숨겨 표시한다.
  - (구분) **HTTP→HTTPS 리다이렉트는 진짜 리다이렉트**다: `http://site`(80)로 가면 서버가 `301/308 Location: https://site`(443)로 되돌려보냄(왕복 발생). "포트 생략→80"(무왕복 가정)과는 다른 개념. → [[HTTP와 HTTPS]]

### 2. nginx 없이 포트 없애기 (도커 포트 매핑)
- `docker run -p 80:8082 myapp` → 호스트 **80포트**를 컨테이너 8082에 연결 → `http://host`로 접속(포트 생략 OK).
- **한계**: 호스트의 80포트는 **한 번에 하나의 프로세스만** 쓸 수 있다 → 앱 하나만 포트 없이 가능. HTTPS(443)·도메인별 분기도 스스로 못 함.

### 3. nginx로 포트 없애기 (리버스 프록시)
- nginx가 **80/443을 listen**하고, `proxy_pass`로 요청을 도커 앱(8082)에 전달([[리버스 프록시]]).
- `server_name`으로 **도메인별 분기**(가상 호스트) → 한 서버에서 `a.com`→앱A, `b.com`→앱B.
- 443에서 **TLS 종료**(인증서 처리) → HTTPS를 nginx가 담당하고 뒤 앱은 평문 HTTP로.

### 4. 언제 포트가 보이나 / 사라지나
| 구성 | 접속 URL | 포트 표기 | 비고 |
|---|---|---|---|
| 앱을 8082에 직접 노출 `-p 8082:8082` | `http://host:8082` | **필요** | 개발/단독 |
| 호스트 80에 매핑 `-p 80:8082` | `http://host` | 불필요 | **앱 1개만**, HTTPS 별도 |
| nginx(80/443) 리버스 프록시 | `http://myapp.com` | 불필요 | **여러 앱·HTTPS·도메인 분기** |

### 5. nginx가 진짜 해결하는 것
- **여러 앱을 하나의 80/443**에서 도메인·경로로 라우팅(포트 매핑만으론 불가 — 80은 하나뿐).
- **HTTPS(TLS) 종료** — 브라우저 기본 https→443을 nginx가 받아 인증서 처리.
- 그 외 캐싱·gzip·보안 헤더·[[로드 밸런서|부하 분산]]([[OSI 7계층|L7]]). (nginx 말고 Caddy·Traefik·클라우드 LB도 같은 역할)

## 코드 / 예시
```nginx
# nginx: 80을 받아 도커 앱(8082)으로 전달
server {
    listen 80;
    server_name myapp.com;          # 도메인별 분기의 핵심
    location / {
        proxy_pass http://127.0.0.1:8082;   # 도커로 띄운 앱
        proxy_set_header Host $host;
    }
}
```
```
브라우저:  http://myapp.com     (포트 안 씀 → 자동 :80)
    │  :80 / :443
    ▼
 [ nginx 리버스 프록시 ]  server_name 분기 · TLS 종료
    │  proxy_pass http://127.0.0.1:8082
    ▼
 [ 도커 앱 :8082 ]   [ 다른 앱 :3000 ] ...
```

## 기억할 문장 / 핵심 포인트
> "포트를 **안 쓰는 것**이 아니라 **기본 포트(80/443)에 있는 것**이다." — nginx는 80을 대신 점유해 뒤로 넘기는 [[리버스 프록시]]일 뿐.
> nginx 없이도 `-p 80:앱포트`면 포트는 사라진다. **nginx가 필요한 진짜 이유는 여러 앱 + HTTPS + 도메인 분기.**
