# 증분 (Incremental)

**한 줄 뜻**: 이전 상태(마지막 시점)와 비교해 **새로 늘어나거나 바뀐 부분(델타)**. "전체(full)"가 아니라 **"달라진 것만"** 다룬다.

**부연**:
- 반대는 **전체(full)** 방식. 증분은 변경분만 처리해 **빠르고 저장·전송이 효율적**이지만, 복원·조립할 땐 **여러 증분을 순서대로 이어붙여야** 한다(체인 의존).
- **증분 백업(incremental backup)** — *마지막 백업(전체든 증분이든) 이후* 바뀐 것만 백업. 가장 작고 빠름. 복원엔 "전체 + 이후 모든 증분"이 필요. ↔ **차등(differential) 백업**은 *마지막 전체 백업 이후* 전부라, 복원은 "전체 + 최신 차등" 2개면 됨.
- **증분 빌드(incremental build)** — 바뀐 소스만 다시 컴파일해 전체 재빌드를 피함(캐싱 기반). → [[툴체인]]
- **증분 업데이트** — 앱·모델에서 변경된 조각만 내려받기. → [[온디바이스 모델 배포]]
