# 핫리로드 (Hot Reload)

**한 줄 뜻**: 앱을 **재시작하지 않고** 바뀐 코드만 **실행 중인 앱에 주입**해 즉시 반영하는 개발 기능. 핵심은 **앱 상태(state)를 유지**한 채 화면·로직이 갱신되는 것 → 개발 피드백 루프가 빨라진다.

**부연** — 헷갈리는 형제 개념들:
- **hot reload** — 변경분만 주입, **상태 보존**. (Flutter는 `main()`/`initState()`를 다시 실행하지 않음) UI 조정·기존 위젯 내부 로직에 적합.
- **hot restart** — 앱을 처음부터 재시작(전체 리빌드보다 빠름), **상태 소실**. 시작 로직·전역 변수·DI 변경 때.
- **live reload** — 파일이 바뀌면 **앱/페이지 전체를 새로고침**, 상태 소실.
- **HMR(Hot Module Replacement)** — 웹 번들러(**Vite**·Webpack)가 **변경된 모듈만 교체**하고 상태 유지(페이지 새로고침 없음). Vite는 파일 감시 → WebSocket으로 변경 모듈만 브라우저에 전송(수십 ms). **React Fast Refresh**는 HMR 위에 React 상태를 똑똑히 보존하는 층.

> ✅ 교차검증 — hot reload(상태 보존·변경분 주입) vs hot restart(상태 소실·빠른 재시작) vs live reload(전체 새로고침) vs HMR(번들러가 변경 모듈만 교체, Vite/React Fast Refresh)의 구분을 Flutter 공식 문서·Vite 문서·복수 자료로 확인.
