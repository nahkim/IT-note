# 스퓨리어스 에러 (Spurious Error)

**한 줄 뜻**: 실제 원인·신호 없이 **우발적으로 발생하는 '가짜/일시적' 오류**. 재현이 잘 안 되고, 대개 무시하거나 재시도로 넘긴다. (spurious = 가짜의·우발적인)

**부연**:
- 대표 예 **스퓨리어스 웨이크업(spurious wakeup)** — 스레드가 신호를 안 받았는데 조건 변수(condition variable) 대기에서 깨어남. 그래서 조건은 `if`가 아니라 **`while` 루프로 재확인**해야 안전하다. → [[동시성 문제]]
- **일시적(transient) 결함**과 통함 — 잠깐 났다 사라짐(순간 네트워크 오류 등) → **재시도(retry)**로 대응.
- 진짜 버그가 아니면서 로그를 어지럽히므로, 원인 없이 반복되면 필터링·완화.
