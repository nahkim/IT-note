# 검증과 확인 (V&V, Verification & Validation)

**한 줄 뜻**: **검증(Verification)** = "제대로 만들었나"(명세대로 구현했나) · **확인(Validation)** = "옳은 걸 만들었나"(사용자의 진짜 필요를 충족하나). Barry Boehm(1979)의 유명한 구분.

**부연**:
- "Building **it** right"(검증) vs "Building the **right** thing"(확인). 명세를 완벽히 만족(검증 통과)해도 **그 명세가 틀렸다면 확인 실패**.
- 검증 수단: 리뷰·[[적대적 리뷰]]·정적분석·단위테스트. 확인 수단: 사용자·인수 테스트, 베타.
- IEEE 1012·ISTQB 등에 표준화된 개념. → [[소프트웨어 테스트]]를 두 목적으로 나눠 보는 큰 틀.
