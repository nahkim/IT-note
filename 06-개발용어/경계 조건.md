# 경계 조건 (Boundary Case)

**한 줄 뜻**: 유효 범위의 **경계값 근처**에서 생기는 문제. 흔한 원인이 **off-by-one**(하나 차이) 실수다.

**부연**:
- **경계값 분석(boundary value analysis)** — 경계 바로 **안·경계·밖**을 테스트. 예: 0~100 범위면 **99·100·101**을 확인.
- 전형적 실수: `<` vs `<=`, 배열 인덱스 끝(마지막/그 다음), 반복문 종료 조건.
- **[[엣지 케이스]]와 사실상 교환 가능**(edge = 경계). **[[코너 케이스]]**는 경계가 여러 개 겹친 것.

**관련**: [[엣지 케이스]] · [[코너 케이스]] · [[소프트웨어 테스트]]
