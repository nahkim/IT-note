# 코너 케이스 (Corner Case)

**한 줄 뜻**: **둘 이상의 조건(파라미터)이 동시에 극단**일 때만 터지는 드문 상황. 다차원 입력 공간의 "모서리(corner)".

**부연**:
- [[엣지 케이스]]와 구분 — **엣지 = 한 파라미터가 극단**, **코너 = 여러 파라미터가 동시에 극단**.
  - 예: (최소 화면 크기) + (최대 폰트) + (오프라인)이 **한꺼번에** 겹칠 때만 깨지는 버그.
- 자주 안 나지만 나면 치명적이고 재현이 어렵다 → 조합 테스트·경계값 분석으로 노려야 함.

**관련**: [[엣지 케이스]] · [[경계 조건]] · [[소프트웨어 테스트]]

> ✅ 교차검증 — corner case(여러 파라미터가 동시에 극단) vs edge case(한 파라미터 극단)의 구분을 educative·복수 자료로 확인.
