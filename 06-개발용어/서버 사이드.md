# 서버 사이드 (Server-side)

**한 줄 뜻**: 코드·처리가 **서버에서 실행**되는 것. 사용자 기기(브라우저)에서 도는 **클라이언트 사이드(client-side)의 반대**.

**부연**:
- **렌더링 관점(SSR vs CSR)**:
  - *서버 사이드 렌더링(SSR)* — 서버가 완성된 HTML을 만들어 보냄. 초기 로딩·SEO에 유리, 서버 부하↑.
  - *클라이언트 사이드 렌더링(CSR)* — 브라우저가 JS로 화면을 그림. 상호작용 풍부하지만 초기 로딩·SEO 불리.
- **보안 관점** — 검증·핵심 로직은 **서버 사이드가 신뢰 경계**. 클라이언트 검증은 우회될 수 있어, 중요한 확인은 반드시 서버에서도.
- 데이터·비밀키·과금 로직 등은 서버 사이드에 둔다.
